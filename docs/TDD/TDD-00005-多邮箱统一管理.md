## 1. 文档目的

本 TDD 描述"多邮箱统一管理"功能的完整技术实现细节。

重点回答：

- 如何在**完全不改造现有 Outlook 流程**的前提下，接入标准 IMAP 邮箱
- 如何通过 `account_type` 字段实现**按账号类型路由分发**
- 如何对 `accounts` 表进行**幂等增量迁移**（不破坏旧数据）
- 新增的 `imap_generic.py` / `providers.py` 的**精确函数签名与行为**
- `repositories/accounts.py`、`controllers/emails.py` 的**最小改造点**
- 前端表单如何**动态切换** Outlook / IMAP 两种模式
- `imap_password` 等敏感字段的**加解密策略**

## 2. 设计原则与硬约束

### 2.1 现有 Outlook 流程零改动

- `services/graph.py`、`services/imap.py`（XOAUTH2）：**完全不动**
- Outlook 的 Graph → IMAP XOAUTH2 回退链：**完全不动**
- 现有 API 的 URL 与基本响应结构：**完全不动**
  - `GET /api/emails/<email_addr>`
  - `GET /api/emails/<email_addr>/extract-verification`
  - `POST /api/emails/delete`
  - `GET /api/accounts`、`POST /api/accounts`、`GET /api/accounts/export`、`POST /api/accounts/export-selected`

### 2.2 DB 约束处理

- `accounts.client_id` / `accounts.refresh_token` 为 `NOT NULL`：**不做 ALTER COLUMN**
- IMAP 账号入库时，上述字段使用占位空字符串 `''`
- 迁移只做 `ADD COLUMN`（幂等），不做 `DROP COLUMN` / `RENAME`

### 2.3 IMAP 行为限制

- IMAP 账号**不执行远程删除**：delete 接口直接返回错误响应
- **定时刷新（scheduler）只处理 Outlook**：查询必须过滤 `account_type='outlook' OR account_type IS NULL`
- IMAP 连接统一使用 `IMAP4_SSL`（端口 993，SSL）

## 5. DB 迁移设计（Schema v2 → v3）

### 5.1 版本常量更新

```python
# outlook_web/db.py
DB_SCHEMA_VERSION = 3  # 原为 2
```

### 5.2 新增列的 DDL

在 `init_db()` 中，紧接现有幂等 `ADD COLUMN` 块之后，新增以下幂等迁移：

```python
# --- Schema v3：多邮箱支持 ---
cursor.execute("PRAGMA table_info(accounts)")
columns = [col[1] for col in cursor.fetchall()]

if "account_type" not in columns:
    cursor.execute(
        "ALTER TABLE accounts ADD COLUMN account_type TEXT DEFAULT 'outlook'"
    )
if "provider" not in columns:
    cursor.execute(
        "ALTER TABLE accounts ADD COLUMN provider TEXT DEFAULT 'outlook'"
    )
if "imap_host" not in columns:
    cursor.execute("ALTER TABLE accounts ADD COLUMN imap_host TEXT")
if "imap_port" not in columns:
    cursor.execute(
        "ALTER TABLE accounts ADD COLUMN imap_port INTEGER DEFAULT 993"
    )
if "imap_password" not in columns:
    cursor.execute("ALTER TABLE accounts ADD COLUMN imap_password TEXT")
```

> **注意**：`PRAGMA table_info(accounts)` 已在前面的代码块中执行过一次（用于检测 group_id 等旧列）。为避免重复查询，应将上述检测合并到同一个列名列表变量中。具体做法：将现有代码中的
> ```python
> cursor.execute("PRAGMA table_info(accounts)")
> columns = [col[1] for col in cursor.fetchall()]
> ```
> 合并为一次查询，统一检测所有列（含旧列 + 新列）。

### 5.3 imap_password 加密迁移

在现有 `migrate_sensitive_data()` 中，补充对 `imap_password` 的迁移逻辑：

```python
# 在 migrate_sensitive_data() 内，处理完 password/refresh_token 之后追加：
rows = conn.execute(
    "SELECT id, imap_password FROM accounts WHERE imap_password IS NOT NULL AND imap_password != ''"
).fetchall()
for row in rows:
    val = row["imap_password"]
    if val and not is_encrypted(val):
        encrypted = encrypt_data(val)
        conn.execute(
            "UPDATE accounts SET imap_password = ? WHERE id = ?",
            (encrypted, row["id"]),
        )
```

### 5.4 schema_version 写入

在迁移成功后的版本写入逻辑中（现有代码已有），`DB_SCHEMA_VERSION=3` 会自动被写入 `settings` 表的 `db_schema_version`，无需额外改动。

```python
def _resolve_imap_folder(
    mail: imaplib.IMAP4_SSL,
    candidates: list,
) -> Optional[str]:
    """
    按顺序尝试 SELECT 候选文件夹名，返回第一个成功的文件夹名。
    若全部失败，返回 None。
    """
    for folder_name in candidates:
        try:
            status, _ = mail.select(folder_name, readonly=True)
            if status == "OK":
                return folder_name
        except Exception:
            continue
    return None
```

| 要点 | 具体实现 |
|---|---|

```python
            else:
                line = f"{acc['email']}----{imap_pwd}----{provider}"
            imap_groups.setdefault(provider, []).append(line)

    parts = []
    if outlook_lines:
        parts.append("# === Outlook 账号 ===")
        parts.extend(outlook_lines)

    for provider, lines in imap_groups.items():
        label = MAIL_PROVIDERS.get(provider, {}).get("label", provider)
        parts.append(f"\n# === IMAP 账号（{label}）===")
        parts.extend(lines)

    return "\n".join(parts)
```

> 现有导出接口的"二次验证"（export_verify_tokens 机制）**保持不变**。

## 10. 错误处理策略

### 10.1 IMAP 错误码与场景

| 场景 | error_code | 用户提示（脱敏） |
|---|---|---|
| 认证失败（授权码错误/过期） | `IMAP_AUTH_FAILED` | "IMAP 认证失败，请检查授权码是否正确" |
| 服务器不可达/端口错误 | `IMAP_CONNECT_FAILED` | "IMAP 服务器连接失败，请检查网络或服务器配置" |
| 指定文件夹不存在 | `IMAP_FOLDER_NOT_FOUND` | "邮箱文件夹不存在，请联系管理员" |
| SSL 握手失败 | `IMAP_CONNECT_FAILED` | 同上 |
| 其他未知错误 | `IMAP_UNKNOWN_ERROR` | "邮件获取失败，请稍后重试" |

### 10.2 错误脱敏要求

在 `imap_generic.py` 中，所有 `except` 块必须：
1. 不将 `imap_password` 包含在日志或返回的 error 字符串中
2. 使用 `sanitize_error_details(str(e))` 处理异常信息（复用 `outlook_web/errors.py`）
3. 对外返回结构统一为 `{"success": False, "error": "<脱敏消息>", "error_code": "<CODE>"}`

### 10.3 IMAP 错误不影响 Outlook 账号

由于路由分发在 controller 层，IMAP 错误只影响当前 IMAP 账号的请求，不会传播到其他 Outlook 账号的调用链。

## 11. 安全设计

### 11.1 imap_password 加解密

| 操作 | 函数 | 时机 |
|---|---|---|
| 入库前加密 | `encrypt_data(imap_password)` | `add_account()` 中，在 INSERT 之前 |
| 读取后解密 | `decrypt_data(account["imap_password"])` | `load_accounts()` 中，在返回前 |
| 迁移加密 | `encrypt_data()` + `is_encrypted()` | `migrate_sensitive_data()` 中补充逻辑 |

复用现有 `outlook_web/security/crypto.py` 中的 `encrypt_data` / `decrypt_data` / `is_encrypted`，不引入新的加密方案。

### 11.2 传输安全

- 所有 IMAP 连接使用 `imaplib.IMAP4_SSL`（TLS，端口 993）
- 不使用 `IMAP4`（明文）

### 11.3 导出脱敏

导出文件包含明文 `imap_password`（已在 `load_accounts()` 解密），属于敏感操作。现有的"导出二次验证（export_verify_tokens）"机制覆盖此场景，**不需要额外改动**。
