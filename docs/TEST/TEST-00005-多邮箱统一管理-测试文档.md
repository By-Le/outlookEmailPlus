## 1. 测试概述

### 1.1 测试目标

本测试文档覆盖"多邮箱统一管理"功能的完整验证，确保：

1. **新功能正确性**
   - DB Schema v3 迁移幂等、不破坏旧数据
   - IMAP 通用服务能正确认证、拉取邮件、返回统一格式
   - 按 `account_type` 路由分发逻辑准确
   - IMAP 账号的敏感字段加解密正确
   - 定时刷新任务只处理 Outlook 账号

2. **Outlook 流程零回归**
   - 现有 Graph API → IMAP XOAUTH2 回退链不受影响
   - 现有批量导入格式（4 段）仍正常
   - 现有邮件删除、验证码提取、账号导出不受影响

3. **安全约束满足**
   - IMAP 密码在 DB 中以加密形式存储
   - 错误消息中不暴露 imap_password
   - 导出二次验证机制不受影响

### 1.2 测试范围

**包含：**
- 数据库迁移（单元/集成）
- providers.py 纯单元测试
- imap_generic.py 单元测试（mock IMAP 连接）
- Repository CRUD 集成测试
- Controller API 集成测试（mock 服务层）
- Scheduler 查询过滤验证
- Outlook 全量回归（集成）
- 手动端到端测试

**不包含：**
- 真实 IMAP 服务器连通性测试（需要外部网络）
- 前端自动化 E2E 测试（Selenium/Playwright）
- 性能压测

### 1.3 测试分类

| 类型 | 工具 | 覆盖层 |
|---|---|---|
| 单元测试 | `unittest` + `unittest.mock` | providers.py / imap_generic.py |
| 集成测试 | Flask test client + 临时 SQLite DB | Controller / Repository / DB |
| 回归测试 | 现有测试套件（不改动） | 所有 Outlook 功能 |
| 手动测试 | 浏览器 + curl | 前端交互 / 真实 IMAP |

## 2. 测试策略

### 2.1 核心测试原则

1. **隔离 IMAP 网络调用**：所有涉及 `imaplib.IMAP4_SSL` 的测试均通过 `unittest.mock.patch` 模拟，不发起真实网络请求。
2. **真实 DB 操作**：Repository 和 Controller 测试使用临时 SQLite 文件（`_import_app.py` 已处理），验证实际 SQL 行为。
3. **最小改动原则**：新增测试类，不修改现有测试文件，确保回归测试基线不变。
4. **按 account_type 分支测试**：每个 Controller 路由测试都要包含 Outlook 和 IMAP 两个分支。

### 2.2 Mock 策略

| 被 Mock 对象 | 路径 | 用途 |
|---|---|---|
| `imaplib.IMAP4_SSL` | `outlook_web.services.imap_generic.imaplib.IMAP4_SSL` | 模拟 IMAP 连接成功/失败 |
| `imap_generic.get_emails_imap_generic` | `outlook_web.controllers.emails.get_emails_imap_generic` | Controller 路由分发测试 |
| `imap_generic.get_email_detail_imap_generic` | `outlook_web.controllers.emails.get_email_detail_imap_generic` | 验证码提取测试 |
| `graph.get_emails` | `outlook_web.services.graph.get_emails` | Outlook 回归 mock |
