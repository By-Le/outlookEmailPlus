## 风险与注意事项

| 风险点 | 应对措施 |
|---|---|
| DB 迁移失败 | 迁移前打印备份提示（现有机制已有）；迁移只做 ADD COLUMN，可安全回滚至 DB 备份 |
| IMAP 连接阻塞 | `socket.setdefaulttimeout(30)` 设置全局超时；IMAP 错误不影响同分组 Outlook 账号 |
| imap_password 明文泄露 | 所有 except 块调用 `sanitize_error_details()`；入库时必须先 `encrypt_data()` |
| 旧 Outlook 流程回归 | 每个阶段结束后必须回归测试 Outlook 邮件读取 + 验证码提取 + 定时刷新 |
| 前端请求体不兼容 | Outlook 分支发送旧格式，向后兼容；后端按 `provider` 字段是否存在做区分 |
